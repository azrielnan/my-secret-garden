# 简历项目表述参考

以下内容仅描述仓库中已经实现的功能，可直接按目标岗位取用。

## 项目名称

**纸人煞：中式民俗 ARG 网页解谜游戏**  
个人项目｜HTML5 / CSS3 / JavaScript / Canvas / Web Audio / GitHub Pages

## 一句话简介

独立设计并实现五章连续的中式民俗 ARG 网页游戏，以浏览器原生能力构建跨场景解谜、非线性叙事与跨章状态持久化体验。

## 简历版要点

- 使用原生 HTML、CSS 与 JavaScript 完成五章网页游戏，设计从旧站检索、源码线索、Canvas 显影到拖拽和键盘组合的多媒介 ARG 交互。
- 基于 `localStorage` 构建跨章状态模型，持久化玩家结局、遗物、道藏碎片与文本回声，并将其映射到后续叙事、场景和终局分支。
- 将第三至第五章重构为“中心枢纽 + 花瓣式探索区”，以遗物和双钥匙机制收束主线，降低线性任务感并保留叙事节奏。
- 使用 Canvas 实现噪点显影、动态频谱、可点击场景绘制与轨迹交互；使用 Web Audio 提供罗盘等轻量反馈音效。
- 通过静态托管部署到 GitHub Pages；补充脚本解析与跨章节链接检查，修复章节跳转、证据链与旧版流程校验冲突。

## 面试展开稿

项目的难点不是单个谜题，而是让“线索发现、叙事解释、状态变化、下一章反馈”保持一致。我把跨章状态拆分为主线结局、可选调查效果与 ARG 收藏三层：主线保证任意正常路线可通关，可选探索只丰富人物动机，收藏内容提供多周目回声。后续又把第三至第五章调整为枢纽式探索，避免玩家被单一按钮链牵着走。
