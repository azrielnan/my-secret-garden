# 《纸人煞》游戏工程说明

## 定位

《纸人煞》是一个单人 ARG 风格网页解谜游戏。它不把谜题写成显性任务列表，而将线索嵌入旧站、档案、物件、场景状态和浏览器可见的技术细节中。

## 章节与枢纽

| 章节 | 叙事核心 | 探索枢纽 | 主线收束 |
| --- | --- | --- | --- |
| 第一章：纸人煞 | 陈五行与苏婉的纸人仪式 | 陈宅与里世界 | 罗盘、符纸、木柜 |
| 第二章：阴司债 | 血契是否出于自愿 | 阴司档案司 | 病历、证词、代签手印 |
| 第三章：忘川渡·昆仑篇 | 青绡的天规与私情 | 三生石 | 玉簪、青丝、旧帕 |
| 第四章：归墟·烛龙 | 因果失衡与烛龙闭眼 | 归墟之门 | 司命台三线与星落池三段记忆 |
| 第五章：道劫·八卦归元 | 情是否应纳入大道 | 太极图 | 六门试炼与离火归元 |

## 脚本职责

- `js/main.js`：旧站基础互动与页面状态。
- `js/ritual.js`：第一章罗盘、符纸、木柜与章节过场。
- `js/exploration-expansion.js`：可选“洞神探幽”调查场景；不作为主线硬门槛。
- `js/arg-field-recorder.js`：玩家记录、线索整理与跨章回声。
- `js/arg-expansion-v4.js`：ARG 热点、复制事件、终端认证、道藏与场景增强。
- `js/narrative-restructure.js`：第三至第五章的花瓣式探索结构和旧逻辑兼容层。

## 状态约定

- `wuxing.chapter*`：章节通关、分支和跨章结局卡。
- `zhirensha.arg.v4`：遗物、道藏碎片、私人回声。
- `zhirensha.narrative.restructure`：第三至第五章的枢纽推进状态。
- `zhirensha.explore.*`：可选调查的证物与效果。

清除站点的本地存储即可从零测试另一条完整路线。

## 验证建议

```powershell
node --check games/zhirensha/js/arg-expansion-v4.js
node --check games/zhirensha/js/narrative-restructure.js
```

此外应从第一章结局依次验证：第二章终端认证、第二章判词写入、第三章遗物开启昆仑、第四章双钥匙问心、第五章终局回声。
